# useIntersectionObserver

A custom React hook designed to track the visibility and intersection changes of multiple DOM elements simultaneously using the native `IntersectionObserver` API. It abstracts away observer setup, lifecycle cleanups, and offers high optimization via a stable mutable ref approach for its callback metrics.

## Features

* **Multi-Element Tracking**: Observe an array of HTML elements efficiently under a single `IntersectionObserver` instance.
* **Stable Callbacks Cache**: Leverages a `useRef` cache pattern so changing dynamic inline callbacks won't cause the underlying observer to tear down and recreate.
* **Granular Lifecycle Hooks**: Distinct lifecycle hooks for entering visibility (`onIntersect`), exiting visibility (`onNoIntersect`), beginning observation (`onObserved`), and cleaning up (`onUnobserved`).
* **Native Configuration Mapping**: Full pass-through configurations for native properties: `root`, `rootMargin`, and `visibilityThreshold`.

## Configuration Options (`OPTIONS`)

  type={{
    observationElements: {
      type: "HTMLElement[]",
      description: "Array of DOM elements to monitor for intersection updates.",
      default: "[]",
    },
    onIntersect: {
      type: "(entries: IntersectionObserverEntry[], observer: IntersectionObserver) => void",
      description: "Callback fired when an observed element enters the visibility threshold range.",
      default: "undefined",
    },
    onNoIntersect: {
      type: "(entries: IntersectionObserverEntry[], observer: IntersectionObserver) => void",
      description: "Callback fired when an observed element leaves the visibility threshold range.",
      default: "undefined",
    },
    onObserved: {
      type: "(element: HTMLElement) => void",
      description: "Lifecycle hook executed immediately when an element begins observation.",
      default: "undefined",
    },
    onUnobserved: {
      type: "(element: HTMLElement) => void",
      description: "Lifecycle hook executed immediately when an element is unobserved or disconnected.",
      default: "undefined",
    },
    visibilityThreshold: {
      type: "number | number[]",
      description: "A single number or an array of numbers indicating what percentage of target visibility should trigger the callbacks.",
      default: "0",
    },
    rootMargin: {
      type: "string",
      description: "Margin around the root element. Serves to grow or shrink the bounding box of the root.",
      default: "'0px'",
    },
    root: {
      type: "Element | null",
      description: "The container element acting as the viewport boundary for testing intersections.",
      default: "null",
    },
  }}
/>

## Exported Values

The hook executes entirely via side-effect configurations and callbacks; it returns `void`.
